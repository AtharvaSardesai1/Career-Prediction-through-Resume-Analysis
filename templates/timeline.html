System workflow

Step 1
Login / Signup

This is done through our Flask App setup which is integrated with MYSQL Database. New User has to create his/her account. Once the account is created users can Login using their credentials and which will stored securely in hashed form in our database.

Step 2
Basic User details and Resume submission

User has to fill some basic details like age, location, years of experience, etc which are required by our model for salary prediction. Also user has to submit his/her Resume in .txt or .pdf format.

Step 3
Resume parsing and Preprocessing

The resume parsing is done with the help of Natural Language Processing (spcay library). If the resume format is .pdf then PyMuPDF library extracts the text from PDF. Text is cleaned (symbols,links,etc removed) to increase the accuracy of the prediction model.

Step 4
Job Category Prediction

Job category prediction classifies text (e.g., resumes) into predefined categories using a TF-IDF vectorizer to convert text into numerical features. These features are passed through a OneVsRestClassifier with a K-Nearest Neighbors (KNN) model for each category. The model predicts the most relevant categories based on the input text, commonly used in resume screening and job recommendation systems.

Step 5
Resume Tone Prediction

Tone prediction classifies the tone of text (e.g., resumes or messages) using a DistilBERT model. The text is tokenized and processed using Hugging Face's tokenizer. The model predicts the tone by analyzing the text's sentiment returning a label (e.g., Formal, Confident, Informal) and a confidence score. This is commonly used to assess the tone of written content, such as resumes or emails.

Step 6
Skills and Courses recommendation

The user is given a recommendation of skills and courses according to his/her predicted domain. The courses are found using Web scraping and mapped to Domains using python dictionary.

Step 7
Salary Prediction

The Salary prediction model is built using a Random Forest Regressor. It preprocesses the data by encoding categorical features and standardizing numerical ones . The model is trained and evaluated on the dataset using cross-validation and metrics like MSE, MAE, and RÂ². Hyperparameter tuning is performed with GridSearchCV to optimize the model. The best model is saved and used to predict salaries for new candidates. Additionally, a decision tree from the Random Forest is visualized to interpret feature importance and model decisions.

Step 8
Career Path Prediction

We construct a career graph using networkx and Graph Convolutional Networks (GCNs) with PyTorch Geometric (PyG) for career path recommendations. Career nodes represent job roles, with edges indicating relationships. Node features, such as programming languages and experience level, are used for training. A two-layer GCN model is defined to predict career paths, and the graph is converted into a PyG Data object for processing and training.
